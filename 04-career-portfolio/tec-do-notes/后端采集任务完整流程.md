# 后端采集任务完整流程

你截图的页面是 “Collect by URL”，对应后端的 link batch 接口。整个流程是这样的：

1.  前端调用 `/api/v1/influencer/collection/realtime/link`，提交 URL 列表 + 平台
    
2.  RealtimeCollectionService.submit\_link\_batch\_task() 创建任务记录（状态 `submitted`），然后 dispatch Celery 异步任务
    
3.  Celery worker 执行 `process_collection_task`，把任务提交到 MATL 采集服务，状态变为 `crawl_processing`
    
4.  Celery Beat 每 30 秒执行 `sync_matl_task_status_job`，轮询 MATL 进度，采集完成后状态变为 `data_processing`
    
5.  Celery Beat 每 10 秒执行 `sync_data_warehouse_status_job`，检查数仓入库进度，达到 90% 阈值（且 flink 实时场景需要连续 2 次 count 不变）后标记 `completed`
    
6.  任务完成时调用 `_send_task_completed_email()` 发送邮件通知
    

## 你阻塞的原因

任务一直卡住，最可能的原因是：

-   Celery Worker 没启动 → `process_collection_task` 根本没执行，任务停在 `submitted`
    
-   Celery Beat 没启动 → 状态轮询的定时任务不会触发，任务卡在 `crawl_processing` 或 `data_processing`
    
-   MATL 服务不可用 → 提交失败或者采集一直没完成
    

你可以先查一下数据库里任务当前的状态：

```sql
SELECT task_id, status, matl_task_id, error_message, created_at, updated_at
FROM kol_realtime_collection_task
ORDER BY created_at DESC
LIMIT 5;
```
